# AI 智能批阅机项目交接说明

日期：2026-06-24

## 1. 当前已确认方向

项目要做一套 AI 智能批阅系统，帮助老师批阅纸质试卷和作业。

当前采用方案 C：

**AI 自动识别 + 老师确认模板。**

也就是先上传正确答案卷，系统自动识别题目、答案区、分值和批改位置；老师确认和修改模板后，再用这个模板批改学生卷。

## 2. 第一阶段范围

第一阶段只做核心识别和批阅能力验证。

优先做：

- JPG、PNG、PDF 导入。
- 正确答案卷识别。
- 题号、答案区、答题区、批改位置识别。
- 试卷模式下每题分值识别和确认。
- 学生卷识别。
- 选择题、判断题、填空题判分。
- 老师人工复核。
- 输出批改 PDF。

暂不做：

- 小程序。
- 家长端。
- 学生端。
- 长期学习分析。
- 错题知识点归因。
- 真实打印机直连。
- 主观题自动评分。

## 3. 两种批改模式

### 作业批改

- 只判断对错。
- 输出对号和错号。
- 不强制计算分数。

### 试卷批改

- 需要确认每题分值。
- 计算每题得分。
- 汇总总分。
- 输出每题得分和总分。

## 4. 关键技术路线

第一阶段技术路线：

- 桌面软件：Electron + React + TypeScript。
- 核心识别服务：Python + FastAPI。
- OCR 主引擎：PaddleOCR。
- OCR 轻量备选：RapidOCR。
- 图像处理：OpenCV。
- PDF 处理：PyMuPDF。
- 本地数据：SQLite。
- 后期模型增强预留：Qwen-VL、MiniCPM-V、GOT-OCR2.0。

关键要求：

- 不直接整张试卷丢给 OCR 后判分。
- 必须先做图像矫正、模板对齐、区域裁剪，再做小区域识别。
- 所有 AI 结果必须支持老师复核。
- 所有模型能力必须可替换。

## 5. 学生身份绑定思路

后续要支持 Excel 或 CSV 导入班级名单。

真实批阅时，老师可先剔除未交作业、缺页严重或暂不批阅的学生，然后按剩余名单顺序整理纸质试卷。

第一阶段身份绑定不是重点。当前优先把识别和批阅能力跑通。

## 8. 下一步建议

下一步应进入实施计划阶段，先做一个最小识别验证版本：

1. 搭建 Python 识别服务。
2. 导入一张真实试卷图片。
3. 做旋转、裁切、去阴影等预处理。
4. 调用 PaddleOCR 识别文字和位置。
5. 输出识别框和文字结果。
6. 评估真实样本识别效果。
7. 再决定模板编辑界面和 PDF 输出怎么实现。
