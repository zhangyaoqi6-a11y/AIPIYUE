# AI 智能批阅机项目交接说明

日期：2026-06-24

## 1. 当前已确认方向

项目要做一套 AI 智能批阅系统，帮助老师批阅纸质试卷和作业。

当前采用方案 C：

**AI 自动识别 + 老师确认模板。**

也就是先上传正确答案卷，系统自动识别题目、答案区、分值和批改位置；老师确认和修改模板后，再用这个模板批改学生卷。

## 2. 第一阶段范围

第一阶段只做核心识别和批阅能力验证。

优先做：

- JPG、PNG、PDF 导入。
- 正确答案卷识别。
- 题号、答案区、答题区、批改位置识别。
- 试卷模式下每题分值识别和确认。
- 学生卷识别。
- 选择题、判断题、填空题判分。
- 老师人工复核。
- 输出批改 PDF。

暂不做：

- 小程序。
- 家长端。
- 学生端。
- 长期学习分析。
- 错题知识点归因。
- 真实打印机直连。
- 主观题自动评分。

## 3. 两种批改模式

### 作业批改

- 只判断对错。
- 输出对号和错号。
- 不强制计算分数。

### 试卷批改

- 需要确认每题分值。
- 计算每题得分。
- 汇总总分。
- 输出每题得分和总分。

## 4. 关键技术路线

第一阶段技术路线：

- 桌面软件：Electron + React + TypeScript。
- 核心识别服务：Python + FastAPI。
- OCR 主引擎：PaddleOCR。
- OCR 轻量备选：RapidOCR。
- 图像处理：OpenCV。
- PDF 处理：PyMuPDF。
- 本地数据：SQLite。
- 后期模型增强预留：Qwen-VL、MiniCPM-V、GOT-OCR2.0。

关键要求：

- 不直接整张试卷丢给 OCR 后判分。
- 必须先做图像矫正、模板对齐、区域裁剪，再做小区域识别。
- 所有 AI 结果必须支持老师复核。
- 所有模型能力必须可替换。

## 5. 学生身份绑定思路

后续要支持 Excel 或 CSV 导入班级名单。

真实批阅时，老师可先剔除未交作业、缺页严重或暂不批阅的学生，然后按剩余名单顺序整理纸质试卷。

第一阶段身份绑定不是重点。当前优先把识别和批阅能力跑通。

## 8. 当前已完成的最小识别验证

已经完成一个 Python 脚本版最小 OCR 验证链路：

1. 读取 `参考资料` 中的真实试卷图片。
2. 做基础图像增强和预处理。
3. 调用 PaddleOCR 识别整页文本框和文本内容。
4. 输出 OCR JSON 到 `data/outputs/minimal-ocr/`。
5. 输出带 OCR 框的调试图片到 `data/outputs/minimal-ocr/`。
6. 根据两张真实样本写入阶段记录。

两张样本识别结果：

- `微信图片_20260616093037_3804_37.jpg`：识别出 217 个文本框。
- `微信图片_20260616093039_3805_37.jpg`：识别出 229 个文本框。

重要结论：

- 整页 OCR 可用于观察识别能力和问题分布。
- 整页 OCR 不能直接作为最终判分依据。
- 背面透字、阴影、折痕、横置页面、多栏混排会明显干扰 OCR。
- 下一步必须做页面方向矫正、模板对齐、题区裁剪和小区域 OCR。

## 9. 环境注意事项

当前项目路径 `F:\智能批阅机` 含中文。Paddle 底层在中文模型缓存路径下可能报 JSON 解析错误，因此 `scripts/setup-env.ps1` 已做处理：

- 学生试卷图片和输出仍放在项目目录中。
- Paddle、PaddleX、HuggingFace、ModelScope 等模型缓存会自动切到 `F:\aipiyue_cache`。
- 临时 home 会自动切到 `F:\aipiyue_home`。

当前 `PaddleOcrEngine` 默认关闭：

- 文档方向分类。
- 文档去畸变。
- 文本行方向分类。
- MKLDNN/oneDNN。

这样是为了保证第一阶段 CPU 本地验证链路稳定跑通。后续如果要恢复这些能力，应单独验证性能和兼容性。

## 10. 下一步建议

下一步不要做完整桌面界面，建议继续做模板和题区级识别：

1. 增加页面方向检测和手动旋转参数，优先解决横置样本。
2. 增加预处理输出最大边长配置，降低 CPU OCR 调试耗时。
3. 基于答案卷或老师框选，先做一版手工模板 JSON。
4. 按模板裁剪 3 到 5 个小题区域。
5. 比较小区域 OCR 与整页 OCR 的稳定性。
6. 为选择题、判断题、填空题分别设计最小规则判分输入格式。
