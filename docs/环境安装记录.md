## 3. 项目环境变量

已创建：

- `.env`
- `.env.example`
- `scripts/setup-env.ps1`

关键变量：

- `OCR_ENGINE=paddleocr`
- `OCR_LANG=ch`
- `PADDLE_DEVICE=cpu`
- `XDG_CACHE_HOME`
- `PADDLE_HOME`
- `PADDLE_PDX_CACHE_HOME`
- `HF_HOME`
- `MODELSCOPE_CACHE`

为什么要设置这些变量：

- Paddle 默认会写入用户目录下的 `.cache`。
- PaddleX 默认会写入用户目录下的 `.paddlex`。
- 如果项目路径是纯英文，OCR 和模型缓存固定在 `data/cache`，便于本地部署、备份和排查问题。
- 当前项目路径 `F:\智能批阅机` 含中文，Paddle 底层在中文模型缓存路径下可能报 JSON 解析错误。因此 `scripts/setup-env.ps1` 会自动把模型缓存切到 `F:\aipiyue_cache`，把临时 home 切到 `F:\aipiyue_home`。
- 学生试卷图片、预处理图、OCR JSON、调试框图仍然保存在项目目录中，不上传云端。
- 当前已设置 `PADDLE_PDX_DISABLE_MODEL_SOURCE_CHECK=True`，用于减少 PaddleX 每次启动前的模型源连通性检查等待。

## 4. 验证命令

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\verify-env.ps1
```

当前验证结果：

```text
paddle 3.3.1
paddleocr import ok
cv2 4.10.0
pymupdf 1.27.2.3
fastapi import ok
```

验证时会出现一个 ccache 警告：

```text
No ccache found.
```

这个警告不影响当前 OCR 识别验证。ccache 主要用于加速 C/C++ 扩展重新编译，第一阶段可以暂不处理。

OCR 真实样本验证时还发现：

- PaddleOCR 3.7.0 默认会加载文档方向、去畸变、文本行方向等额外模型。当前最小验证已在代码里关闭这些非必要模型。
- PaddleOCR 在 CPU 上默认启用 MKLDNN/oneDNN，本机组合下会出现 `ConvertPirAttribute2RuntimeAttribute not support`。当前 `PaddleOcrEngine` 默认传入 `enable_mkldnn=False`。
