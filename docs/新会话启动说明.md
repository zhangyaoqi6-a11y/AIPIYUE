## 2. 新会话需要优先确认的事项

新会话开始后，应先做以下检查：

1. 检查当前目录是否是 `F:\智能批阅机`。
2. 执行 `git status --short --branch`，确认本地仓库状态。
3. 执行环境验证命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\verify-env.ps1
```

4. 确认 `paddle`、`paddleocr`、`cv2`、`pymupdf`、`fastapi` 都能正常导入。
5. 确认 `参考资料` 文件夹中有真实试卷图片。

## 4. 推荐的第一批代码任务

建议新会话按这个顺序推进：

1. 新建一个 Python 后端目录，例如 `backend/`。
2. 写一个图片预处理脚本，读取 `参考资料` 中的图片。
3. 输出预处理后的图片到 `data/outputs/`。
4. 写一个 PaddleOCR 识别脚本。
5. 把识别结果保存为 JSON。
6. 生成一张带 OCR 框的调试图片。
7. 根据真实样本效果，写一份 `docs/阶段记录/第一阶段识别验证记录.md`。
